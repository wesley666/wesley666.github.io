安卓开机速度分析方法

本文最后更新于：2026年3月7日 晚上

一般拿到机器，我们统计开机时间，主要看如下几个时间段分布：

  1. 开机按键时间、亮屏时间
  2. uboot 启动时间
  3. 内核启动后到 bootanim 退出时间

Init会记录一些启动时间信息到属性（部分如下）：

ro.boottime.init

Time after boot in ns (via the CLOCK_BOOTTIME clock) at which the first stage of init started.

ro.boottime.

Time after boot in ns (via the CLOCK_BOOTTIME clock) that the service was first started.

bootchart分析方法

bootchart 是通过采集 / proc 中的数据，将系统启动过程记录下来，通过 host 端工具将其解析为可视化图表的工具；

会收集如下信息：

  1. CPU 负载：/proc/stat
  2. 磁盘吞吐：/proc/diskstats
  3. 各个进程的启动过程：/proc/$pid/stat 需要注意，上述 bootchart 开始录制是在 init.rc 所以主要分析的是 android 启动阶段（对于 linux 来讲则是 init 进程后的信息）；

在存在 enabled 文件的情况下，在 init.rc 启动时开始，bootcomplete 时结束：

抓取流程

1、adb shell touch /data/bootchart/enabled 2、adb shell reboot

3、 进入/data/bootchart执行 tar -zcf bootchart.tgz *

4、使用adb pull 命令将文件拷贝出来

注意：不需要抓取时，记得把enabled标志删除

linux分析数据（没有服务器也可以使用wsl2来分析）

对比两个 bootchart 图

android 7.0 代码还添加了新功能，执行 compare-bootcharts.py 脚本可以对比获取的两个 bootchart 的进程启动结束时间点，从而得到对应进程之间的耗时差距。

将所要比较的两个 bootchart 的压缩包放在两个不同的目录 base_bootchart_dir,

exp_bootchart_dir. 执行如下命令:

system/core/init/compare-bootcharts.py base_bootchart_dir exp_bootchart_dir

FAQ

  1. android 全部版本都支持么？ 在 androidM 之前此工具需要重新 build 打开才能使用，在之后变成现在的样子；
  2. 以前有使用过可以添加参数的 bootchart，主要是设置 timeout 时间，现在不行了？ 从 code 实现来看，android高版本中内置的工具是不支持时间配置的，不过可以做个简单的处理，可以修改init.rc 中的 属性值，改为手动停止。

FAQ

1、在搭载 API 级别 27 及更低级别的设备和模拟器上，如果没有足够的可用内存或内存过于碎片化，您将收到以下消息：Atrace could not allocate enough memory to record a trace。如果发生这种情况，并且您没有捕获到完整的数据集，请关闭后台进程或重新启动设备或模拟器。

2、设置了persist.debug.atrace.boottrace开机启动boottrace服务的话，要及时停止，否则可能影响perfetto执行

3、限制：记录跟踪事件是在加载persist属性之后开始的，因此不会记录之前发出的跟踪事件。一些服务（如vold、surfaceflinger和servicemanager）受此限制的影响，因为它们是在加载persist属性之前启动的。Zygote初始化和从Zygote fork的进程不受影响。

Perfetto UI

3、启用perfetto_trace_on_boot服务

注意：这个属性会在下一次开机时重置，如果要在下下次继续抓取trace，需要重新设置。

4、结果会生成在/data/misc/perfetto-traces/boottrace.perfetto-trace
