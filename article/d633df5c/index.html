二进制基础知识

本文最后更新于：2025年3月2日 晚上

将补码（Two’s Complement）转换为十进制的方法主要有以下几种。以下是详细的推导和示例：

方法 1：标准公式法

补码的十进制值由符号位和数值位共同决定：

  • 公式：
    [
    \text{十进制值} = (-1)^{\text{符号位}} \times \left( \text{数值位的二进制值} \right)
    ]
  • 步骤：
    1. 判断符号位：最高位为符号位（0表示正数，1表示负数）。
    2. 计算数值位：
      • 正数：符号位为0，直接按二进制转十进制。
      • 负数：符号位为1，需将补码转换为原码，再计算十进制值。

示例（8位补码）

  • 正数补码：0010 1101
    符号位为0，直接计算数值位：
    (2^5 + 2^3 + 2^2 + 2^0 = 32 + 8 + 4 + 1 = 45)
    结果为 +45。

  • 负数补码：1110 1011
    符号位为1，需转换：

    1. 取反补码：0001 0100
    2. 加1：0001 0101（十进制为21）
    3. 添加符号：结果为 -21。

方法 2：逐位权重计算法

直接根据二进制位的权重计算，符号位单独处理：

  • 公式：
    [
    \text{十进制值} = - (\text{符号位} \times 2^{n-1}) + \sum_{i=0}^{n-2} (\text{bit}_i \times 2^i)
    ]
    • (n)：总位数（如32位、8位等）。

示例（8位补码 1110 1011）

  1. 符号位：最高位为1，权重为 (-2^7 = -128)。
  2. 数值位：剩余7位 110 1011，计算：
    [
    64 + 32 + 8 + 2 + 1 = 107
    ]
  3. 合并结果：
    [
    -128 + 107 = -21
    ]

方法 3：补码原码转换法

通过补码与原码的关系计算：

  • 步骤：
    1. 负数补码：符号位为1时，补码 = 原码取反 + 1。
    2. 恢复原码：将补码减1后取反，得到原码，再计算十进制值。

示例（8位补码 1110 1011）

  1. 减1：1110 1010
  2. 取反：0001 0101（十进制21）
  3. 添加符号：结果为 -21。

特殊情况：最小值

对于n位补码，最小值为 (-2^{n-1})，此时数值位全为0，符号位为1。
示例（8位补码 1000 0000）：
[
\text{十进制值} = -2^7 = -128
]

总结

方法 适用场景 示例
标准公式法 正负数通用，直接计算 1110 1011 → -21
逐位权重计算法 编程实现或快速计算 1110 1011 → -21
补码原码转换法 手动验证或理解原理 1110 1011 → -21
最小值特殊处理 符号位为1且数值位全0的情况 1000 0000 → -128

关键点

  1. 补码的符号位决定了正负。
  2. 负数的补码需通过取反加1转换为原码。
  3. 最小值的补码形式为符号位1 + 数值位全0。
